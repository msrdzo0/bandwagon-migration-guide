## 什么是搬瓦工机房迁移？能解决什么问题？

搬瓦工的机房迁移，官方叫 **Migrate to another DC**（DC 是 Data Center 的缩写）。简单说就是：把你当前 VPS 上的全部数据，原封不动地搬到另一个地理位置的服务器上去。

适合用这个功能的场景：

- 当前机房延迟突然变高，换个线路更好的机房
- IP 被封了，换机房同时也能换 IP（注意这和"购买新 IP"不是一回事）
- 你的业务受众位置变了，比如从面向欧美换成面向亚太
- 想测试不同线路，找到最适合自己网络运营商的机房

## 迁移前要知道的三件事

### 1. IP 会变

从一个机房迁到另一个机房，VPS 的 IP 地址会换成目标机房的新 IP。如果你的 VPS 上挂了域名解析，迁完之后要去更新 DNS 记录。

有一个细节：如果你从 A 机房迁到 B，再从 B 迁回 A，IP 大概率恢复成最初那个——除非原来那个 IP 在你走了这段时间被别人用了。

### 2. 数据完整保留

迁移过程中，VPS 上所有文件、数据库、配置文件、已安装的软件，全部原样保留。这一点和"重装系统"完全不同。重装是清空重来，迁移是打包搬家。

### 3. 迁移时间和中断

根据 VPS 里数据量的多少，迁移时间从十几分钟到最长几小时不等。大多数普通情况在 15 分钟内完成。迁移末尾阶段 VPS 会有几秒到几分钟的短暂关机，完成后会发邮件通知。

## 详细操作步骤：如何在 KiwiVM 里迁移机房

以下步骤适用于支持机房迁移的套餐（不支持迁移的套餐，在步骤 2 中会看到提示信息，点击后直接显示"不支持"）。

**第一步：登录搬瓦工后台**

进入搬瓦工官网，点击顶部导航的 Services → My Services，找到你要迁移的 VPS 实例，点击旁边的 **Open KiwiVM Control Panel** 按钮，进入该 VPS 的 KiwiVM 控制面板。

**第二步：找到迁移功能入口**

KiwiVM 左侧菜单里，有一个 **Migrate** 区块，点击 **Migrate to another DC**，右侧会显示可切换的目标机房列表。

**第三步：选择目标机房**

列表里会显示你当前所在的机房（Current location），以及所有可切换的目标机房。根据你的网络运营商和延迟需求选一个，然后点 **Confirm on next step**。

**第四步：确认并开始迁移**

确认页面会提示你：迁移过程中数据会保留，IP 会变，VPS 末尾会短暂关机。确认没问题后点 **Start Migration**，等待完成即可。迁完后系统会发邮件告知你新的 IP 地址。

## 迁移时常见问题

**问：迁机房之前需要备份数据吗？**

搬瓦工的迁移机制不会删除数据，但任何涉及数据操作的步骤，养成提前快照备份的习惯总是对的。好消息是搬瓦工的快照功能是免费的，在 KiwiVM 里一键搞定。

**问：IP 被封了，能靠迁机房换 IP 吗？**

能。迁到新机房之后会分配一个新 IP。但需要注意：如果你的 IP 已经被封，搬瓦工系统会检测到这个状态，在部分情况下会限制你使用迁移功能——这时候可以联系客服工单处理。

**问：KVM 套餐迁到 CN2 机房流量为什么变少了？**

这是搬瓦工的一个定价规则。KVM 普通套餐切换到 DC3（CN2 GT）机房时，流量会按 0.33 的倍率折算，也就是实际能用的流量变成原来的三分之一。如果你对流量有要求，建议直接购买 CN2 GIA-E 套餐，切换到 CN2 系列机房不会有流量折算的问题。

**问：迁移之后原来的服务、端口、SSH 设置都还在吗？**

都在。迁移相当于把整台 VPS 的磁盘内容原样搬到新机房，所有文件系统、配置文件、已安装的服务都会保留。唯一需要手动更新的，就是域名的 DNS 解析记录（因为 IP 变了）。

**问：CN2 GIA-E 套餐能切换到香港机房吗？**

不行。香港机房是搬瓦工体系里最高端的节点，只有购买香港套餐本身才能进入，CN2 GIA-E 套餐的可切换列表里不包含香港 CN2 GIA 机房。
